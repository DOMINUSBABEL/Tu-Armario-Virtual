# Contributing to DY(DressYourself)

We love pull requests from everyone. By participating in this project, you agree to abide by the code of conduct.

## Development Process
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes.
4. Update documentation if necessary.
5. Create a Pull Request (PR) referencing your issue or idea.

## UI/UX Guidelines
- Please review `VISION.md`. All UI components must use `Compose Multiplatform`.
- Do not introduce platform-specific UI unless absolutely required for native functionality.
